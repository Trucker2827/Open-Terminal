# v0.3.20 — Fix: READ FULL works again

v0.3.17 changed READ FULL to lead with a search-crawler user-agent, which made
sites that gate crawlers fail to load — so READ FULL stopped returning full
text for most articles. Fixed: the normal browser fetch runs first (as it did
before), and the crawler/archive fallbacks only kick in when that comes up
short. Verified the full article body loads again.
