# v0.3.12 — FRED Analytics fixed

## Fixed
- **FRED Analytics now returns data.** The panel was sending the dataset command
  twice (e.g. "money_supply money_supply m1"), so FRED got a bad series/date and
  returned HTTP 400 — which, because the request URL contains "api_key=", was
  mislabeled as "FRED API key not configured." Now fixed; M1/M2/M3, yield curve,
  credit rates, stress, sentiment and PCE all return data.
- **API key no longer leaks into error text.** FRED error messages embedded the
  full request URL (including your api_key); the key is now redacted to `***`.
