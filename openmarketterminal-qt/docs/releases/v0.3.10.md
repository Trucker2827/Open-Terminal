# v0.3.10 — Local AI models that actually use the terminal's data + web browsing

Local/offline models (Ollama) now stop making up stale answers and instead call
the terminal's live tools — and the in-app AI can browse the web for current
information.

## Fixed

- **Local models no longer hallucinate live data.** A local model (e.g. llama 3.3
  via Ollama) asked "what's AAPL trading at?" used to invent a stale price from its
  training data. The cause: tool discovery was a two-step dance (find the tool, then
  call it) that weak models skipped. Local models now receive a curated set of
  essential live-data tools **directly** (quotes, news, SEC financials, candles,
  symbol lookup, web search, …) and are instructed to **never answer a price or
  recent event from memory — always call a tool**. The same query now returns the
  real, current quote. Premium cloud models (Claude) are unchanged.

## New

- **Web browsing for the in-app AI.** Two new tools the assistant can call:
  - **`web_search`** — searches the web (DuckDuckGo) and returns the top results
    (title, link, snippet) for current, general information beyond the built-in
    finance data.
  - **`web_fetch`** — opens a page and returns its readable text.
  - Both run through the app's **bundled Chromium engine, headless** — no extra
    install, no API key, and pages that need JavaScript still render.

## Notes

- Web tools use **DuckDuckGo** with no API key; results are filtered to drop ad
  links. They are read-only.
- Local-model capability still scales with the model; premium Claude remains the
  recommended engine for deep analysis. This release is about making local models
  *reliably reach live data* rather than guessing.
