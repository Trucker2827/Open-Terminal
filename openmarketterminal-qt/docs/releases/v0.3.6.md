# v0.3.6 — Ownership data (Insiders / Institutions / Politicians) + read-only Observer

Read-only research additions: who's buying and selling — corporate insiders,
institutions, and members of Congress — plus a headless market Observer surfaced
across the CLI, GUI, and the in-app AI. Everything new here is **read-only**; no
trading path is added.

## New

- **Ownership screens (Research):**
  - **Insiders** — corporate insider trades (SEC Form 4): who bought/sold, transaction
    code, shares, and price, by ticker.
  - **Institutions** — institutional 13F holdings (SEC Form 13F-HR): top positions by
    value, with manager + total-portfolio summary.
  - **Politicians** — US Congress trades by ticker (via AInvest): member, party, state,
    buy/sell, amount range, and filing date.
  - Open from the Component Browser (Cmd+K) or the command bar; all three are also
    available as **Dashboard widgets** (Insiders, Institution 13F, Politicians).
- **Observer** — read-only BTC/ETH observation journal (daily trade-lesson, weekly
  review, alerts): a dockable GUI panel, an `observe latest|week|alerts` CLI command,
  and a `get_observations` tool for the in-app AI.

## Improved

- **Navigation cleanup** — the top bar now leads with **Strategies** and **Notebooks**;
  the internal engines (Strategy Lab, Pricing Lab, Quant Lab, Workflows) moved out of the
  primary tabs but remain fully available in the Component Browser and command palette.
  Command-palette names now match the on-screen labels.
- **AInvest** added to Settings → Credentials (`AINVEST_API_KEY`) — required for the
  Politicians screen/widget to load data; stored in the OS keychain like every other key.
- QuantLib copy corrected to describe a **local** pricing engine.

## Fixed

- **SEC Form 4 / 13F parsing.** The existing insider/13F tools returned "success" with
  empty payloads (Form 4 owner/shares/price were null; 13F "holdings" were table header
  strings). Rewritten on `edgartools` to return real rows; a parse failure now surfaces
  a clear "parser unavailable" state instead of a misleading blank.

## Notes

- **Politicians needs an AInvest API key** (Settings → Credentials). Without one the
  screen shows "AInvest not configured."
- Insiders/Institutions use **SEC EDGAR** (free, public-domain). Politicians uses AInvest
  (commercial) — congress data is filed up to ~45 days after the trade.
- Searching Congress trades **by politician name** is not in this release (the AInvest
  endpoint is ticker-keyed); planned as a follow-up on a full-dataset source.
