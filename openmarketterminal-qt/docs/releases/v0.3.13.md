# v0.3.13 — Economics data sources: more panels work + keys configurable

## Fixed — panels that returned errors now fetch data

A shared bug (same class as FRED Analytics in v0.3.12) made several Economics
panels send their command twice, corrupting the API request:

- **ECB** now returns FX rates, HICP, M3, etc. (was a malformed SDMX key → 404).
- **Eurostat** now returns industrial production / retail / trade by country
  (the country was landing in the wrong dimension → HTTP 413).
- **Trading Economics** arg handling fixed (still needs an API key — see below).
- **OECD**: the panel was sending a frequency code ("A") the script rejected;
  it now sends "annual"/"quarter". *(Known limitation: OECD's SDMX endpoint
  still returns HTTP 422 for these queries — the script's OECD query format is
  stale against OECD's current API. Tracked as a separate fix; OECD will not
  return data yet.)*

I audited all 32 Economics panels for this command-doubling class — the four
above (plus FRED Analytics) were the only ones affected; the rest were correct.

## Added — API keys are now configurable in the app

BLS, BEA, EIA, WTO and Trading Economics require API keys, but those keys had
no field in **Settings → Credentials** and weren't passed to the data scripts —
so there was no way to make these sources work, and the panels told users to
"set an environment variable" that was never read. Now:

- All five keys appear in **Settings → Credentials** and are injected into the
  data scripts (same path as the FRED key).
- The panels' messages point to Settings → Credentials instead of an env var.

Add a key there (free signups: data.bls.gov, bea.gov, eia.gov, developer.wto.org,
tradingeconomics.com/api) and those panels will fetch.

## Verification note

These are GUI panels with file-local dataset tables and no unit-test harness, so
the fixes were verified by reproducing each script's exact corrected invocation
directly (ECB and Eurostat confirmed returning data; OECD confirmed past the
frequency gate) rather than by automated tests.
