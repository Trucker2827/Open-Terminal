# v0.3.32 — Quant Lab made real, and bridged to the terminal

## AI Quant Lab honesty overhaul
- `run_backtest` now runs a **real** qlib TopkDropout simulation (costs included,
  equal-weight-of-universe benchmark) instead of returning hardcoded metrics.
- `get_factor_analysis` computes the **real** daily information coefficient vs
  realized labels instead of canned numbers.
- Trained models persist to disk (`~/.qlib/openterminal_models`) — training,
  prediction, backtest, IC, and feature importance now work across processes.
- Dense-input models (linear, LSTM, GRU, Transformer…) are trainable: NaN
  feature columns are filled for models that cannot tolerate them, and the
  linear default config no longer aborts the OLS estimator.
- mlflow ≥ 3 no longer aborts every training run.
- New `screen` command: cross-sectional top/bottom model ranks with an IC
  caveat in every response.

## High Frequency Trading module
- binance (HTTP 451 geo-restricted) replaced: coinbase default, kraken and
  gemini in the panel, `BTC/USD` symbols.
- Fixed a cross-venue crash: kraken order-book levels carry a third element.

## CLI
- New `quant` command family: `quant list`, `quant run <module> <command>
  ['{json}']`, `quant script <module>` — all 24 Quant Lab modules (Deep Agent
  included) runnable from the terminal with the same scripts, venv routing,
  and environment as the GUI. No running app required.
- Bridge client fix: responses from slow tools are no longer truncated
  ("no header terminator") — reads until server close with a total deadline.

## Terminal ↔ Quant Lab bridge (new)
- **Offline reads** (`terminal_data.py`): analytics wrappers accept
  `{"source":"terminal","symbol":"BTC-USD"}` and run on the terminal's own
  stored tick history instead of pasted arrays or yfinance.
- **Live reads** (`terminal_bridge.py`): minimal read-only client for the
  app's local MCP bridge (candles, order books).
- **Crypto qlib dataset** (`crypto_dataset.py`): builds a qlib bar dataset
  from the terminal's own coinbase/kraken ticks; `OPENTERMINAL_QLIB_DATA` /
  `OPENTERMINAL_QLIB_FREQ` point the whole ML stack at it.
- **Model→engine evidence** (`signal_publisher.py`): published model signals
  with **earned trust** (trailing rank-IC must be statistically significant);
  the scalp gate journals them and can only be blocked by a trusted
  contradicting signal via explicit opt-in. Untrusted models are noise by
  construction; the deterministic executor remains sole authority.

## Engine
- Realized volatility end-to-end: the serve daemon attaches ambient vol to
  every decision snapshot; the blind-context slot reserved (and empty) all
  through v4 is now populated; scoring rows carry vol for
  difficulty-normalized cohorts.
- Scalp gate: `observed_move_sigma` + `--min-move-sigma` noise floor (default
  1σ, fails open without vol) — moves indistinguishable from ambient noise no
  longer look like signals.
- New advisory tools: `spot_calibrator.py` (online settlement-trained
  probability calibrator with a market-mid baseline it must beat) and
  `edge_ic_report.py` (IC of predicted edge vs realized move over the
  resolved journal).

## Data screens
- GOVT Data crash fixes, AKShare removal, key-less FRED→DBnomics fallback
  (carried from v0.3.31).
