# v0.3.11 — FRED economics data works + US-first defaults

## Fixed

- **FRED (Federal Reserve) economics data now displays.** The FRED panel fetched
  the series correctly but then dropped every data point — it expected text
  values while the data arrives as numbers — so it showed "no data" even on a
  good fetch. Fixed; FRED series (S&P 500, GDP, CPI, yields, …) now render.
- **Clearer FRED key guidance.** The panels used to say *"Set FRED_API_KEY
  environment variable"* — but the app stores the key in **Settings →
  Credentials → "FRED"**. The empty-state and error messages now point there
  (free key still at fred.stlouisfed.org).

## Changed — US-first throughout

The app is a US/global terminal, but a few defaults still leaned on its Indian
origins. Now US-first everywhere a default is shown:

- **Strategies / backtests:** bare symbols are treated as **US** tickers (a bare
  "AAPL" was being turned into the Indian "AAPL.NS"), and the default symbol is
  **AAPL** instead of RELIANCE — so US backtests work out of the box. Non-US
  markets still work when you type the exchange suffix (e.g. `RELIANCE.NS`,
  `7203.T`, `VOD.L`).
- **AI assistant examples** (place-order, option-chain, company lookup,
  watchlist) now use US examples (AAPL/MSFT/SPX) instead of Indian ones.
- Removed unused Indian scan-universe presets and tidied related leftovers.

## Notes

- The world market-hours widget still lists India alongside the US/UK/Japan/China
  sessions — that's global coverage, not a default.
