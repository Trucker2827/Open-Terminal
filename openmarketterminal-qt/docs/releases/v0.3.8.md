# v0.3.8 — Chat sessions that name themselves + editable names

Quality-of-life for the AI Chat: new conversations now name themselves after
your first message instead of getting a random codename, and you can rename or
delete any conversation from a right-click menu. Plus a curated library of 100
example prompts and a code-signing fix.

## New

- **Chats are named after your first message.** New conversations used to get a
  random codename ("Atlas Brief D99E"). Now, the moment you send your first
  message, the session is titled from that message (trimmed to a readable
  length) — so the sidebar reads "What's the DCF value of AAPL" instead of a
  random word pair. A conversation you've manually renamed is never overwritten.
- **Rename / Delete from a right-click menu.** Right-click any conversation in
  the sidebar for **Rename** and **Delete** (the existing Rename/Delete buttons
  still work too). Renaming opens a simple text prompt.
- **100 example prompts** (`docs/ai-chat-example-prompts.md`) — a grounded,
  copy-paste library of questions the in-app AI can actually run by calling the
  terminal's tools, grouped into 15 categories (quotes, SEC financials,
  valuation, ownership, news, crypto, macro, government data, M&A, options,
  portfolio, technicals/quant, reports/notebooks, prediction markets, and
  "do-it-for-me" workflows).

## Fixed

- **App no longer breaks its own code signature.** The exchange/markets cache
  was being written *inside the app bundle* (next to the bundled script), which
  invalidated the macOS code seal on first run ("a sealed resource is missing or
  invalid"). The cache now goes to the app's per-user data directory, so the
  signed bundle stays intact.

## Notes

- Auto-naming only applies to **new** conversations going forward; existing
  conversations keep their current names (rename them from the right-click menu
  if you like).
