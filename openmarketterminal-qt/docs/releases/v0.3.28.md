# v0.3.28 - Local Trading Cockpit and Research Runtime

- Adds a local crypto automation cockpit with clearer daemon, account, signal,
  and execution-state visibility.
- Expands the terminal CLI control plane and daemon health/runtime reporting for
  local research and bounded trading workflows.
- Connects Research Lab screens to the active local evidence, calibration, and
  strategy-workflow data instead of static templates.
- Adds an optional local-app lock toggle while preserving local-only credential
  storage.
- Strengthens Kalshi/crypto signal collection, paper-trading records, and
  deterministic regression coverage.

This release keeps user credentials, account data, journals, and local model
configuration on the user's machine. No API keys or secrets are included in
the release artifacts or source tree.
